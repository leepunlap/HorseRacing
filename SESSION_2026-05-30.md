## 1. Real-time infrastructure (Socket.IO + Redis)  вЂ” `2427155`, `40f5aa3`

- Replaced the raw `/ws/progress` WebSocket with a **Socket.IO server backed by
  Redis** (`AsyncRedisManager`), mounted in front of FastAPI as `app:asgi`.
  This lets **out-of-process jobs (cron scripts) stream to the UI** without
  importing the app вЂ” they publish to Redis, the server fans out to browsers.
- `utils/emit.py` вЂ” fire-and-forget log emitter for scripts.
- `status.py` вЂ” structured **process/task status layer**: producers
  (`process_up/heartbeat/task_start/step/done/error`) publish to a `hr:status`
  Redis channel; an in-app consumer folds events into a registry and forwards
  to browsers. `GET /api/status` returns the snapshot.
- Client pinned to WebSocket-only transport to silence polling-upgrade noise.
- Dep note: pinned `h11==0.14.0` / `wsproto==1.2.0` so python-socketio coexists
  with httpx. Installed **system-wide** (the systemd unit has `ProtectHome=true`,
  so user-site packages are invisible to it).
